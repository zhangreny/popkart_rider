# 剪刀石头布自动化脚本假设说明

本文记录 `play_caiquan.py` 中依赖的固定前提，以及在“UI 布局不变、图标不变”的情况下仍需关注的可变项。

## 前提假设

- 游戏窗口标题为 `PopKart Client`。
- 目标显示器设备 ID 包含 `DELD1EE`。
- 活动界面、挑战界面、右侧阶段列表、结果弹窗、奖励确认弹窗的 UI 布局保持不变。
- 模板图片保持有效，包括 `entry.png`、`yes.png`、`no.png`、`jiandao.png`、`shitou.png`、`bu.png`、`user-jiandao.png`、`user-shitou.png`、`user-bu.png`。
- 剪刀石头布规则不变：剪刀胜布，石头胜剪刀，布胜石头。
- 阶段规则不变：第 1-2 阶段对方出一张牌，第 3-5 阶段对方出两张牌。

## 可接受的硬编码

在 UI 布局和图标不变的前提下，以下硬编码是合理的：

- 模板文件名。
- 剪刀石头布胜负关系。
- 第 1-2 阶段识别一张对方牌，第 3-5 阶段识别两张对方牌。
- 第 5 阶段胜利后点击“否”，随后点击奖励弹窗的“确定”。
- 右侧阶段区、结果牌区、奖励确认按钮区的窗口比例。
- 阶段高亮、右侧“赢”图标、蓝色按钮的颜色阈值。
- 循环模式每轮完成后重新从入口开始。

## 仍可能需要调整的项目

- `DISPLAY_ID = "DELD1EE"`：如果换显示器、显卡驱动重排显示器，或设备 ID 变化，需要修改。
- `PopKart Client`：如果窗口标题变化，需要修改。
- 等待时间和超时时间：机器卡顿、网络慢、游戏动画延迟时，可能需要调大。
- 图像匹配阈值：如果分辨率、缩放、亮度、抗锯齿效果变化较大，可能需要微调。
- 循环停止方式：当前通过创建 `stop.loop` 停止，脚本会在当前轮结束后退出循环。

## 当前脚本稳定性的关键点

- 点击位置不是绝对坐标，而是通过模板匹配或窗口比例区域计算得到。
- 每一阶段都会读取右侧蓝色高亮阶段，不依赖脚本内部计数盲走。
- 平局不会进入下一阶段，脚本会继续识别当前阶段并重新出牌。
- 第 5 阶段是否胜利通过结果弹窗背后的实际出牌判断，不只依赖右侧阶段是否变化。
- 第 5 阶段胜利后会自动处理奖励确认弹窗，避免卡住下一轮入口。

## 常用运行方式

单次执行：

```powershell
py .\play_caiquan.py
```

循环执行：

```powershell
py .\play_caiquan.py --loop --loop-delay 4
```

最多循环指定轮数：

```powershell
py .\play_caiquan.py --loop --max-runs 10 --loop-delay 4
```

停止循环：

```powershell
Set-Content -LiteralPath .\stop.loop -Value stop -NoNewline
```
